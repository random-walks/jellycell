# Daily time series — tearsheets

Each tearsheet below is auto-generated by `jellycell export tearsheet
notebooks/<name>.py` and renders inline on GitHub — click any file to see
the figures and summary tables for that notebook.

| # | Notebook                                         | Tearsheet                           | Produces                                      |
| - | ------------------------------------------------ | ----------------------------------- | --------------------------------------------- |
| 1 | [`01-explore.py`](../notebooks/01-explore.py)    | [`01-explore.md`](01-explore.md)    | `daily.parquet`, `summary.json`, two plots    |
| 2 | [`02-decompose.py`](../notebooks/02-decompose.py)| [`02-decompose.md`](02-decompose.md)| `decomposed.parquet`, decomposition plot      |
| 3 | [`03-forecast.py`](../notebooks/03-forecast.py)  | [`03-forecast.md`](03-forecast.md)  | `forecast.parquet`, `report.json`, diagnostics|

## Regenerate

```bash
cd examples/timeseries
jellycell run notebooks/01-explore.py   notebooks/02-decompose.py   notebooks/03-forecast.py
jellycell export tearsheet notebooks/01-explore.py
jellycell export tearsheet notebooks/02-decompose.py
jellycell export tearsheet notebooks/03-forecast.py
```

## Things to notice in the tearsheets

- **Artifact-based dataflow.** Notebook 02 loads `artifacts/daily.parquet`
  (written by 01) via `jc.load`. No shared kernel state required — each
  notebook runs in its own subprocess kernel.
- **Matplotlib figures.** `jc.figure(fig=fig)` persists figures as PNG
  under `artifacts/` and the tearsheet embeds them with relative paths so
  GitHub renders them inline.
- **Declared deps propagate cache invalidation.** Changing the noise seed in
  cell `raw` of `01-explore.py` invalidates the downstream cache in
  `02-decompose.py` too (because the Parquet bytes change).
- **JSON summaries.** Cells that call `jc.save(dict, "artifacts/<name>.json")`
  become two-column tables in the tearsheet — see [`03-forecast.md`](03-forecast.md)
  for an example with nested `in_sample.*` and `residuals.*` fields.

## What's synthetic vs. what's real

The data is synthetic (generated inline) so the demo is reproducible without
external downloads. The pipeline is real — swap the `raw` cell for a
`pd.read_csv(...)` over your actual daily data and everything downstream
works unchanged.
